## ファイルエクスポート (File Export)

「システムがファイルとして出力する」データフローを定義するために使用されます: エクスポートスコープ、権限、形式、パフォーマンス、および監査。

適用シナリオ:
- レポートのエクスポート、照合のエクスポート、サードパーティへのバルクデータの配信
- データの保持とアーカイブ、監査証拠の収集

推奨される情報:
- エクスポートスコープ: フィルター条件、フィールドセット、並べ替えルール
- 権限と匿名化: どのロールがエクスポートできるか、機密フィールドの処理
- ファイル形式: CSV/XLSX/JSON、エンコーディング、日付/金額の形式
- パフォーマンス: ページネーション/非同期タスク、ダウンロードリンクの有効期間、同時実行制限
- 監査: エクスポート実行者、エクスポート時間、条件、ファイルのチェックサム

実装の提案:
- エクスポートタスクの状態マシンとエラーレシートを統一します
- 大容量データの戦略 (シャーディング、圧縮、ボリュームの分割、レジューム可能なダウンロード) を明確にします
