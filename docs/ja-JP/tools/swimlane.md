## スイムレーン図 (Swimlane Diagram)

「クロスロール/クロスシステムのコラボレーションプロセス」を表現するために使用され、誰がそれを行うか、どこで実行されるか、引き継ぎポイントはどこかという責任の境界を強調します。

適用シナリオ:
- マルチロールの参加 (顧客/運用/財務/コンプライアンス/サードパーティ)
- マルチシステムのコラボレーション (メインシステム + 外部チャネル + 照合システム)

推奨される構造:
- スイムレーンのディメンション: ロール / システム / 部門 / サービス (いずれかをメインとして選択)
- 引き継ぎポイント: イベント/メッセージ/API呼び出し/ファイルの配信
- 主要な出力: 各スイムレーン段階の入力/出力

実装の提案:
- 各引き継ぎポイントのプロトコル (API、MQ、ファイル、手動操作) を明確にします
- 障害の帰属と補償の責任 (どの当事者が再試行/ロールバック/手動で介入するか) を明確にします
