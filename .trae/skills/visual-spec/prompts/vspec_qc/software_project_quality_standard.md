## 适用行业：software_project（软件研发项目/交付/DevOps）

#### 1. 需求到交付的可追溯性缺失：关键词“追溯/需求/设计/测试/发布/矩阵”
- 问题分类：治理缺失
- 关键词：追溯/需求/设计/测试/发布/矩阵
- 问题描述：缺少端到端追溯会导致变更影响不可控、验收缺证据
- 错误问题示例：需求变更后未更新测试用例与验收标准
- 解决方案：建立追溯矩阵（需求→设计→实现→测试→发布）；变更触发更新；明确负责人；输出可审计证据（链接与版本）
- 方案收益：降低返工与交付风险
- 应用场景：立项、迭代、验收
- 关联质量标准：可追溯性

#### 2. 版本/分支/合并策略不清：关键词“版本/分支/主干/发布分支/合并策略”
- 问题分类：流程缺失
- 关键词：版本/分支/主干/发布分支/合并策略
- 问题描述：分支与发布策略不清会导致上线冲突与回滚困难
- 错误问题示例：热修复如何回合到主干、是否需要打 tag 未说明
- 解决方案：定义分支模型（trunk-based/gitflow）；版本号规则；发布窗口与冻结期；合并准入（代码评审、CI 通过）；tag 与 release note 规范
- 方案收益：提升发布稳定性
- 应用场景：迭代发布、紧急修复
- 关联质量标准：一致性

#### 3. CI/CD 质量门禁缺失：关键词“CI/CD/单测/覆盖率/静态扫描/制品”
- 问题分类：质量控制缺失
- 关键词：CI/CD/单测/覆盖率/静态扫描/制品
- 问题描述：缺少质量门禁会导致缺陷与安全问题进入生产
- 错误问题示例：未要求单测通过即可合并，生产频繁回归
- 解决方案：定义门禁：lint/format、单测、覆盖率阈值、SAST、依赖漏洞扫描、制品签名；失败阻断；门禁豁免审批
- 方案收益：提升缺陷拦截率
- 应用场景：合并、构建、发布
- 关联质量标准：正确性

#### 4. 变更管理与回滚预案缺失：关键词“变更/灰度/回滚/开关/兼容”
- 问题分类：风险控制缺失
- 关键词：变更/灰度/回滚/开关/兼容
- 问题描述：未定义灰度与回滚会导致故障扩大
- 错误问题示例：数据库变更上线后发现错误，但无回滚脚本与兼容策略
- 解决方案：定义变更单字段（范围、风险、回滚）；灰度策略；功能开关；向后兼容（双写/扩字段）；回滚脚本与演练
- 方案收益：降低事故影响面
- 应用场景：上线发布、数据迁移
- 关联质量标准：鲁棒性

#### 5. 依赖与供应链安全缺失：关键词“依赖/漏洞/SBOM/许可证/供应链”
- 问题分类：安全缺失
- 关键词：依赖/漏洞/SBOM/许可证/供应链
- 问题描述：依赖治理不足会引入高危漏洞与许可证风险
- 错误问题示例：直接引入未维护依赖，未做 CVE 扫描
- 解决方案：建立依赖白名单；自动漏洞扫描与升级策略；生成 SBOM；许可证合规检查；制品签名与校验
- 方案收益：降低安全与合规风险
- 应用场景：构建、发布、审计
- 关联质量标准：安全性

#### 6. 环境与配置/密钥治理缺失：关键词“环境/配置/密钥/分级/脱敏”
- 问题分类：安全缺失
- 关键词：环境/配置/密钥/分级/脱敏
- 问题描述：配置与密钥管理不当会导致泄露与环境漂移
- 错误问题示例：密钥写入代码仓库，日志输出敏感配置
- 解决方案：定义环境分层（dev/test/staging/prod）；配置中心；密钥托管；访问控制；脱敏与审计；配置变更审批与回滚
- 方案收益：降低泄露与误配风险
- 应用场景：部署、运维、审计
- 关联质量标准：合规性

#### 7. 缺陷/事件管理与 SLA 不清：关键词“缺陷/事件/等级/SLA/复盘”
- 问题分类：治理缺失
- 关键词：缺陷/事件/等级/SLA/复盘
- 问题描述：缺少事件分级与 SLA 会导致故障响应不可控
- 错误问题示例：线上故障无法定义 P0/P1 口径与响应时限
- 解决方案：定义事件等级与响应/恢复 SLA；值班与升级机制；告警阈值；复盘模板与整改闭环
- 方案收益：提升稳定性与透明度
- 应用场景：运维、客服、研发
- 关联质量标准：可追溯性

#### 8. 发布说明与变更证据缺失：关键词“发布说明/变更记录/审批/审计”
- 问题分类：审计缺失
- 关键词：发布说明/变更记录/审批/审计
- 问题描述：缺少发布说明与审批证据会导致审计不通过与回溯困难
- 错误问题示例：上线内容与影响范围无法追溯到具体需求与 commit
- 解决方案：定义 release note 模板；审批链；关联需求与工单；制品版本与部署记录；可导出审计报告
- 方案收益：提升可审计性
- 应用场景：上线、验收、审计
- 关联质量标准：可追溯性
