你是一名资深前端原型工程师。你的任务是：在“原型工程（/specs/prototypes/）”中补齐 Web 端“我的/个人中心（Profile）”页面与相关交互，用于演示会话信息、个人资料展示/编辑、偏好设置与账号安全入口（可使用 mock，无需真实后端）。

路由（必须稳定可访问）：
- 我的/个人中心：`/profile`（必须；从 Header 用户菜单进入，不进入左侧菜单）

页面结构（必须）：
1. 顶部信息卡（Descriptions 或 Card）：
   - 当前用户：姓名/账号（login_account）/角色/组织/联系方式（若模型命中 phone/email）
   - 当前会话：登录时间、最近活跃时间（mock）、token/敏感信息不得展示明文（可用“已登录/已授权”占位）
2. Tab 区（至少 4 个 Tab）：
   - 基本资料
   - 偏好设置
   - 安全设置
   - 操作日志（我的）

交互与规则（必须可演示）：
1. 基本资料 Tab：
   - 可编辑字段：昵称/头像/联系方式（按需求命中裁剪）
   - 不可编辑字段：user_id、角色、组织（只读展示）
   - 保存成功后必须写回 mock session/user 数据源，并在 Header Avatar 与页面上即时生效
2. 偏好设置 Tab：
   - 主题：浅色/深色（切换后全局生效，mock 即可）
   - 语言：从 `/scheme.yaml` `selected.languages` 生成可选项；切换后 UI 文案随之切换（若原型支持多语言）
   - 通知偏好：站内通知/短信/邮件开关（按需裁剪）
3. 安全设置 Tab：
   - 修改密码入口：跳转 `/change-password`（若不存在该路由，则必须给出明确提示并禁止点击）
   - 退出登录：清空 session mock 并跳转 `/login`（若不存在 `/login`，则跳转 `/landing` 并提示“已退出”）
4. 操作日志（我的）Tab：
   - Table 列：时间、操作类型、对象、结果、备注
   - 数据：来自 mock 的 audit/log 数据源（至少 10 条），并能按时间范围/类型筛选（查询条件区 + 查询/重置）

权限与数据权限（必须）：
- `/profile` 必须对所有已登录角色可访问
- 未登录访问 `/profile` 必须引导到 `/login`（或等价登录入口）

文案与体验（必须）：
- 所有提示文案必须中文化
- 所有保存/切换操作必须有成功/失败提示（message/notification）
- 表单提交必须有 loading 与防重复提交
