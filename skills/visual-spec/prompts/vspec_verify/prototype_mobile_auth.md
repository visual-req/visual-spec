你是一名资深前端原型工程师。你的任务是：在“原型工程（/specs/prototypes/）”中补齐 Mobile 端账号体系（非 SSO）相关页面与交互闭环（可使用 mock，无需真实鉴权）。

适用条件（命中则必须执行）：
- 需求未采用 SSO/OIDC/LDAP 等统一身份接入，且属于“本系统独立登录”形态

路由（前缀必须为 `/m/*`，且必须稳定可访问）：
- 登录：`/m/login`
- 创建账号：`/m/signup`
- 忘记密码：`/m/forgot-password`
- 重置密码：`/m/reset-password`
- 修改密码：`/m/change-password`

页面与闭环（必须可演示）：
1. `/m/login`
   - 账号选择（List/Picker；适配手机交互）
   - 密码输入
   - 登录按钮
   - 链接：去创建账号、忘记密码
   - 登录成功：建立 session mock，跳转到移动端首页入口（例如 `/m/list` 或移动端工作台）
2. `/m/signup`
   - 字段：账号/手机号/邮箱（按需求命中裁剪）、密码、确认密码
   - 成功后：写回 mock 用户数据源，并可回到 `/m/login` 选择新账号登录
3. `/m/forgot-password` → `/m/reset-password`
   - 发送验证码（mock）→ 输入验证码重置密码（校验 + 成功提示 + 回到登录）
4. `/m/change-password`
   - 必须为“已登录可访问”页面
   - 成功后：强制退出（清空 session）并引导重新登录

会话与路由拦截（必须）：
- Mobile 端同样必须受 session mock 保护（未登录访问移动端业务页必须引导到 `/m/login`）
- 必须提供“退出登录”入口（例如移动端个人页或顶部更多菜单）

交互与控件规范（必须）：
- 所有选择类控件必须使用移动端友好的选择器页面/底部弹层（禁止在 mobile 用下拉 Select）
- 表单必须有 loading 状态、防重复提交、错误提示（中文文案）
